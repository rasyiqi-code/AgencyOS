## 2024-05-14 - Redundant Database Queries for Static Content in Layouts
**Learning:** Layout components (like `app/dashboard/layout.tsx`) run on almost every navigation within their route segments. Directly fetching static configuration from the database (e.g. `prisma.systemSetting.findMany`) inside these layouts causes a significant bottleneck and redundant database load, as it circumvents Next.js caching layers if not properly wrapped. The application already possessed an `unstable_cache` wrapped utility (`lib/server/settings.ts -> getSystemSettings`), but it wasn't universally applied in key layouts.
**Action:** Always use existing caching utilities (like `unstable_cache` or dedicated helper functions) when fetching rarely-changing data (like system/agency settings) in high-traffic components like RootLayout or DashboardLayouts. Look out for `findMany` calls fetching constant or configuration data directly inside components.
